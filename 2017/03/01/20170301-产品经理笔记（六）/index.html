产品经理笔记（六）

发表于 2017-03-01   |   分类于 产品经理   |  

什么是MVP

MVP = Minimum Viable Product(最简可行产品)

  • 在MVP实验阶段，为了得到客户的真实反馈，你用最少的资源开发了最简版本的产品，来确认你的想法是否可行
  • 这是一个科学实验，不要使客户产生偏见
  • 快速迭代发布多个MVP，收集更多的反馈信息来筛选哪个版本是最优的产品

产品经理眼中的MVP

  • 大公司对于风险的耐力要强于创业公司
  • 即使你的MVP产品失败了，大公司也有足够的资源生存下去
  • 在大公司，产品经理关系的不应该是资源，而是要关心失败会对品牌带来什么影响
  • 从另一个角度看，失败也可能为公司带来另一种收益

MVP实验的7个步骤

  1. 发现你的问题是什么/你的解决方法是什么
  2. 确认你的设想并寻找风险
  3. 围绕你的设想（assumption）建立可测试的假设（hypothesis）
  4. 确立成功的最小标准
  5. 选择你要进行实验的MVP和你的战略
  6. 执行MVP实验
  7. 评估和学习这次实验

证实你的假设

大家总是假设自己知道所有事情，即使大家心知肚明这是不可能的事。

风险假设

如果这个假设不成立，那么难的产品注定的会失败

  • 根据风险的大小和难易程度做出决策

搭建一个假设（hypothesis）

假设（hypothesis）和设想（assumption）的区别在于，假设是一套可以执行的，有目标群体，有期望值的战略方法，让客户按照既定的路线配合

  • 例子
    • 我们相信（目标群体），会（预测行为），因为（原因）
    • 如果我们（行为），我相信（主题）会（预测行为/结果），因为（原因）
    • 我们相信（主题）有（问题）， 因为（原因），如果我们（行为），这个（标准）就会改进

什么是最小成功标准（minimum criteria for success）

MVP实验一般会有三种结果：1）失败 2）成功 3)进展中（somewhere in the middle），大多数情况下都是以第三种方式结束。所有我们需要建立一个概念（MCS）——最小成功标准来审核是否值得构建产品

  • MCS让实验目的变得清晰
  • 为了设立MCS标准，你需要考虑构建产品的成本和回报
  • 不能被证实的潮流点子不是好点子
  • 确定目标的MCS可能会引起潜在客户的兴趣
  • 计算所有成本，包括开发时间、薪水、广告费用、品牌效应、机会成本
  • 成本必须比回报要小，这样产品才有价值

MVP的种类

  • 邮件MVP

    • 邮件客户端、邮件地址清单、写作技巧
    • 通过邮件观察用户的反馈
    • 尽量人性化，以免邮件被丢进垃圾箱

  • 假按钮（shadow button）MVP

    • 只有文字描述没有实际功能的一个按钮
    • 点击后链接到“功能开发中”
    • 点击量可以反应客户的兴趣

  • 说明视频MVP

    • 用视频阐述新的功能
    • 分教程风格和销售风格
    • 在教程风格的视频里，演示如何使用特殊功能，用视频特效进行展示，即使功能目前尚未实现
    • 在销售风格的视频里，利用虚拟宣传片展示产品的带来的效益

  • 单页面产品主页MVP

    • 介绍产品的功能和特点
    • 单页面风格
  • 礼宾服务MVP
    • 客户开发和MVP的混合
    • 不需要构建任何东西
    • 和客户沟通了解需求
