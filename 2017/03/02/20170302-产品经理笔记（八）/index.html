什么是量化标准（metric）

作为产品经理，你需要通过制定标准来衡量产品是否达标。

  • 反馈循环： 精确的反馈越频繁说明效果越好
  • 量化标准就是一些数字或者方法来描述你的产品的进展情况
    • KPI（Key Performance Indicators）
    • 举例
      • 每月活跃用户
      • 回归用户
      • 应用商城评论
      • facebook或者twitter的流量
    • 根据不同的产品不同的公司会有不同的量化标准
    • 产品是否成功就看量化标准完成的质量

量化标准的分类

  • 成长&活跃量化标准

    • 新的用户
    • 区别只安装app的用户
    • 活跃用户是指真实登入了并执行相关操作的用户

  • 保有量化标准

    • 有多少用户持续使用该产品

  • 交互量化标准

    • 跟着记录用户使用频率
    • 根据不同公司不同产品会有不一样的标准

  • 用户满意度量化标准

    • 用户的满意度
    • 量化有一定难度，但是非常重要

  • 收益量化标准

    • 产品能产生多少效益
    • 月/年收益
    • 投入回报比（cost per acquisition）

如何选择合适的量化标准

  • 探索性量化标准
    • 不是你跟踪的对象，但是分析他可以知道发生了什么
  • 报表式量化标准
    • 长期跟踪的对象，可以确保产品方向的正确性
  • 好的量化标准
    • 易懂性
    • 可以用比率或者等级衡量
    • 相关性
    • 易变性

H.E.A.R.T.量化标准框架

  • Happiness： 用户的满意程度
  • Engagement： 在短时期内用户的投入程度
  • Adoption: 有多少用户使用了产品
  • Retention： 产品的用户黏性如何
  • Task Success： 最重要的是用户使用你的产品
  • 实际运用中应该是A.T.E.R.H.这样的顺序
  • HEART框架非常灵活，可以按需选取量化标准使用

在每个量化标准下，都需要注意3件事

  • 目标Goals： 你其他发生什么
  • 信号Signals： 为什么接近目标你得知道实际要量化的是什么
  • 量化标准Metrics： 你是如何利用目标和信号来表达量化标准
  • 利用Google Analytics

A.A.R.R.R.量化标准框架

  • Accquisition： 获得客户
  • Activation： 用户活跃度
  • Retention： 用户黏性
  • Referral： 用户推荐
  • Revenue： 产品收益
