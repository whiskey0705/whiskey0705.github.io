产品经理笔记（五）

发表于 2017-02-28   |   分类于 产品经理   |  

什么是客户开发（Customer development）

客户开发是一项重要的手段，产品经理通过客户开发获得反馈，来确定构建的产品的方向是否正确。产品经理通过客户开发，来减少风险，把握机会。

客户开发周期

  • 确认： 在这个阶段，你需要和客户面谈确认产品的需求以及需要解决的真正问题
  • 开发第一版： 根据之前客户的需求，构建产品特征
  • 迭代： 在这个阶段需要改进产品，询问客户是否喜欢该产品，哪些特征缺失，哪些功能需要添加

和客户面谈的四种类型

  • 探索面谈（Exploratory interview）

    • 形式自由，帮助客户确定痛点，以及探讨可行的解决方案
    • 以客户的视角谈论产品，观察他们准备如何使用产品，是否愿意付费
    • 提开放式的问题

  • 确认面谈（Validation interview）

    • 你有了一套理论并准备测试，需要通过科学的方法
    • 在谈论最后才说出你的理论，避免先入为主
    • 在描述的时候应该尽量保持客观
    • 让客户自己描述问题所在

  • 满意导向面谈（Satisfaction oriented interview）

    • 找出产品好的和不好的地方
    • 明白客户为什么满意和不满意

  • 效率面谈（Efficiency interview）

    • 通过什么途径可以让产品更好地为目标服务
    • 了解客户如何使用产品以及对产品功能的期待

不同种类的客户开发

客户开发贯穿产品的整个生命周期

  • 未发布的产品

    • 潜在的客户
    • 客户不知道你的产品
    • 比较难接触客户
    • 应该关注痛点和方案的可行性

  • 已发布的产品

    • 存在已有客户
    • 客户知道你的产品
    • 比如容易接触客户
    • 关注满意度、可用性和痛点

如何建立目标客户群

首先确定目标客户是谁，根据产品是否发布，会有不同的目标客户。

  • 未发布的产品： 你仍在确认阶段，大家喜欢这个产品吗，产品是否有市场？

    • 情况1：你脑袋里已经有潜在客户，你将对他们进行开发
    • 情况2：没有目标
    • 问3个问题
      • 给产品预设三类潜在客户
      • 产品解决了什么问题
      • 谁会有这样的问题

    • 3个标准

      • 大小： 市场份额
      • 痛点： 支付形式
      • 可得性（Accessibility）：是否容易接触客户

    • 哪里寻找客户

      • LinkedIn
      • 论坛
      • Twitter
      • 你的竞争对手

  • 已发布的产品：已经有客户基础了，根据反馈选出最合适的目标

    • 如何与客户交流
      • 使用公司的实时聊天系统
      • 通过公司的blog
      • Twitter

如何让客户和你对话

  • 简短：没人会看大篇幅的陌生邮件，理想的长度是4-7句话
  • 人性化
    • 没人喜欢和机器人对话
    • 主动阐明来意
  • 价值体验
    • 让客户觉得对话是有价值的
    • 客户喜欢帮助的感觉
    • 告诉客户他们的反馈对你很重要
    • 物质奖励
    • 告知对方能并非销售
    • 让对方感受到特殊待遇

如何组织邮件内容

  1. 介绍（1-2句），告诉对方你在哪里找到他们的联系方式以及你注意到的问题
  2. 2句话阐述你为什么想交谈
  3. 1句话安排见面时间

如何进行正确的客户面谈

你想要确保得到精确和有用的信息，面谈的主题不是你的产品

  • 不要谈你的解决方式
    • 谈客户遇到的问题
    • 客户可能不知道他们想要（want）的是什么，但是他们无法隐藏需求（need）
  • 不要谈你自己的观点
    • 应该以客户为主
    • 有时可能会迸发出新的想法

  • 创造一个舒适的环境

    • 不要问会使他们紧张或者不舒服的问题
    • 对于反馈信息不要反应过度，不应该消极
    • 尽可能保持中立，不要走批判路线

  • 不要强迫面谈走势，应该引领

    • 让客户谈他们的关注点
    • 这样的方式可以让你得到更多的信息
    • 当你卡住的时候，可以表达你很感兴趣，希望对方多讲一些

  • 5个W

    • 谁是你的客户
    • 他们有什么习惯
    • 他们什么时候需要产品
    • 在什么地方需要用到它
    • 为什么他们需要它

好问题和坏问题

坏问题会让答案带上偏见，好问题可以得到真实的反馈

  • 原则一： 永远问开放性的问题
  • 原则二： 不要问二元性问题（可以用是或者不是回答）
  • 原则三： 不要问假设的问题
  • 原则四： 不要问导向性的问题
  • 原则五： 不要问可能促使对方撒谎的问题

创建一个虚拟角色

创建一个虚拟角色可以更好的呈现相似用户的行为习惯

  • 虚拟角色可以与相似行为的用户产生共鸣
  • 虚拟角色的信息包括
    • 名字、年龄、照片、简介
    • 对他们来说最重要的是产品的哪个特征
    • 他们有什么更进一步的需求

如何确定你收集了足够多的信息

你不可能只依据一个信息源，就做出决定，因为这样会冒很大风险做出错误的产品。客户面谈反馈只是一种信息源

  • 有用的信息源
    • 内部反馈
    • 用户测试数据
    • 网络反馈
    • 分析系统
    • 新闻
    • 市场趋势
    • 观察竞争对手
